Problemas comunes en WordPress

Describo los problemas más frecuentes, su impacto y una estimación realista del tiempo de resolución.

Problemas frecuentes

Cada ítem incluye causa habitual, impacto y tiempo estimado de resolución.

Pantalla blanca (WSOD) / Error fatal

Causas: actualización de plugin/tema, error PHP o límite de memoria. Impacto: sitio inaccesible.

  • Solución típica: activar WP_DEBUG en staging, desactivar plugins por CLI/FTP, restaurar backup si es necesario.
  • Tiempo estimado: 30–120 minutos.

Rendimiento lento / Core Web Vitals bajos

Causas: imágenes sin optimizar, plugins pesados, hosting limitado, scripts bloqueantes.

  • Solución: optimizar imágenes y servir WebP, activar cache (page + object), usar CDN y revisar plugins.
  • Tiempo estimado: 3–12 horas.

Conflictos entre plugins y temas

Causas: APIs incompatibles, hooks mal empleados o versiones de PHP no compatibles. Impacto: funciones rotas, errores en el front.

  • Solución: debug incremental, desactivar y reactivar por grupos, revisar logs y actualizar código en child theme.
  • Tiempo estimado: 1–4 horas.

Problemas tras actualizaciones (core, plugins o tema)

Causas: incompatibilidades o migraciones incompletas.

  • Solución: restaurar desde backup, test en staging, aplicar fixes o versiones compatibles.
  • Tiempo estimado: 1–3 horas.

Sitio comprometido / Malware

Causas: plugins vulnerables, credenciales filtradas o falta de hardening. Impacto: pérdida de datos, SEO penalizado y riesgo legal.

  • Solución: aislamiento, limpieza (scanner + manual), rotación de credenciales, hardening y reporte.
  • Tiempo estimado: 4–40 horas (según alcance del incidente).

Problemas con pagos / checkout

Causas: configuración de pasarelas, SSL, webhooks o inconsistencias en productos. Impacto: pérdida de ventas.

  • Solución: revisar logs de gateway, probar en sandbox, verificar SSL y endpoints de webhook.
  • Tiempo estimado: 2–8 horas.

Entregabilidad de emails

Causas: mala configuración SMTP, falta de SPF/DKIM o IP compartida en blacklist. Impacto: emails transaccionales no llegan.

  • Solución: configuración SMTP profesional, SPF/DKIM/DMARC y uso de provider (SendGrid/Mailgun).
  • Tiempo estimado: 1–3 horas.
